# AGENTS  

Этот файл служит указателем: полный текст правил и обязательных процедур хранится в отдельных документах и не дублируется здесь.

- Базовый протокол работы, включая требования к поддержанию RoadMap и memory-bank, рабочий цикл и завершающий ритуал, описан в `Protocol.md` (см. разделы «Базовый протокол…» и пункты 1–14). Перед стартом задач сверяйтесь с ними.
- Единственным источником истины по шагам, проверкам и состоянию проекта остаётся `RoadMap.md`; быстрый протокол действий находится в его разделе «Быстрый протокол действий».
- Процедуры проверок и gate-чеков берите из `RoadMap.md`: раздел «Протокол проверки» для пошаговых проверок и «Gate-чек переключения майлстоунов» перед сменой майлстоуна.

Если при чтении этих ссылок обнаружатся обновления требований, следуйте им — указанные документы имеют приоритет.

# КАК работать над задачами

при старте новой задачи создава создавай новую ветку из main 
Всегда работай в этой ветке для данной фичи. 

В конце сессии дай короткий отчёт: список коммитов.

Не допускай разветвления верхушек ветки на разные коммиты при общей базе. Коммиты одной дорожной карты должны быть последовательны в одной ветке до её PR, либо начаты с начала. Всегда проверяй это. Если по каким-то причинам это произошло, то немедленно сообщи пользователю.
