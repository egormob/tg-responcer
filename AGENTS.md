# AGENTS

Этот файл содержит короткое напоминание о ключевых правилах работы и ссылки на источники истины. Полные тексты см. в `Protocol.md` и разделе «Быстрый протокол действий» в `RoadMap.md`.

## Базовые принципы
- `RoadMap.md` — единый источник истины о задачах, прогрессе и проверках; перед каждой задачей сверяйтесь с ним и актуализируйте записи по ходу работы.
- Соблюдайте рабочий цикл: микроплан, партии изменений ≈150 файлов, обязательные проверки после каждой партии, фиксация прогресса и отдельные коммиты для подпунктов дорожной карты.
- В конце каждой сессии описывайте выполненные действия, пройденные проверки и следующую цель; следите за линейностью истории ветки.

## Быстрый протокол действий
- Перед работой проверяйте ветку/статус, перечитывайте `RoadMap.md`, памятки в `memory-bank/` и требования по `OPENAI_MODEL` / `OPENAI_PROMPT_ID`.
- При инцидентах сначала выполняйте протокол `memory-bank/troubleshooting-protocol.md`.
- Проверки обязательны; при падении заведите или обновите «ЗАДАЧУ-ТОРМОЖЕНИЕ: проверки не закрыты», устраните проблему, перезапустите проверки и только после успеха переходите дальше.
- Фиксируйте в журнале прогресса номер шага/майлстоуна и ссылки на проверки; для внешних проверок создавайте задачу «ПРОВЕРКА» и отмечайте переход к ней в журнале.
- Поддерживайте чистоту истории (`git status`, `git diff --check`) и отчёты на русском языке с прогресс-баром.

## Ограничения
- Не добавляйте бинарные файлы в репозиторий без прямого разрешения; артефакты проверок храните вне git.
