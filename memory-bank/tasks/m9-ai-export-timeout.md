# Задача: зафиксировать влияние двойного `/export` на ответы ИИ

## Что сделать
1. **Подготовка.**
   - Убедиться, что воркер задеплоен с версиями от 2025-11-25 (router c auto-resolve + `[dialog-engine][ai_fallback]`).
   - Поднять `wrangler tail` с фильтром `ai|dialog-engine|router` и включить JSON (`--format json`).
2. **Сценарий.**
   - В админском чате отправить `/export`, дождаться начала формирования файла.
   - В течение 5 секунд повторить `/export` (нарушив кулдаун) минимум дважды.
   - Пока кулдаун отвечает «Экспорт формируется…», с основного пользовательского аккаунта отправить обычное сообщение (без команд).
3. **Сбор данных.**
   - Зафиксировать лог `wrangler tail` (должен содержать `[dialog-engine][ai_fallback]` и `[ai][timeout]` с `queueDetails`).
   - Выполнить `curl https://<worker>/admin/diag?q=ai-queue -H 'x-admin-token: …'` сразу после сценария, сохранить JSON.
   - Сохранить скриншот/видео переписки (админ и пользователь) с временными отметками.
4. **Отчёт.**
   - Создать запись в `reports/REPORT-ai-export-timeout-YYYYMMDD.md` с ссылками на tail, diag и скриншоты.
   - Обновить `memory-bank/diagnostics.md` (секция 10) результатом: подтвердилось ли влияние `/export` на ответы ИИ.

## Данные и артефакты
- `wrangler tail` (желательно ≥60 секунд, JSON).
- Ответ `/admin/diag?q=ai-queue` (JSON).
- Скриншоты Telegram (админ, пользователь).

## Быстрый старт
```sh
npm run task:checkout m9-ai-export-timeout
```
